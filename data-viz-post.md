# 数据可视化图表Prompt生成技能

将科技/商业新闻素材转化为晚点LatePost风格的数据图表，可输出 image-2 prompt、直接生成图片，或制作固定 900×1200px 的 HTML 数据卡片。

## 工作流程

### 阶段零：触发与输入入口

当 agent 检测到本 skill 被激活后，先判断用户是否已经提供素材：

- 已提供链接、PDF、截图、正文、表格或结构化数据：直接进入阶段一
- 未提供素材：请用户输入链接或上传 PDF/截图/数据文件
- 用户已明确指定设计方向、图片数量、输出模式或分辨率：沿用用户指定，不重复询问
- 用户没有指定设计方向：默认读取并遵循 `assets/style-reference-1.md`

### 阶段一：素材研读与数据整理

1. **读取素材**：处理用户提供的链接、PDF、截图、正文或数据表，提取事实、数字、时间、主体与来源。
2. **补充检索**：当数据可能过时、缺失或存在口径冲突时，用 web_search 验证并补足；实时数据必须记录时间戳。
3. **数据结构化**：整理成时间线、对比表、关键指标、排行、占比、分组或事件链。
4. **单位核对**：为每个数字记录单位、币种、时间口径和统计口径；缺失或不确定时先确认，不能直接写入 prompt 或图表。

### 阶段二：切分标题与小节内容

先不要直接写 prompt 或生成图片。必须先给用户一个可修改的内容切分方案：

```markdown
## 内容切分预览

1. [标题/小节名]
   - 核心信息：
   - 关键数据：
   - 建议图表：
   - 数据来源：
2. ...
```

**确认规则**

- 明确询问用户是否需要修改标题、小节顺序、内容重点、图表数量或数据口径
- 用户要求修改时，只更新切分方案并再次确认
- 用户确认切分方案后，进入阶段三
- 如果用户一开始就给出了完整切分方案，可快速核对后继续

### 阶段三：图表规划与模式选择

根据数据特点规划图表序列，常用图表类型：

| 数据类型 | 推荐图表 |
| -------- | -------- |
| 时间序列增长 | 折线图/面积图 |
| 两方对比 | 分组柱状图/双数字对比 |
| 多指标对比 | 左右两列对比表 |
| 里程碑事件 | 双线时间线 |
| 占比/份额 | 水平条形图 |
| 单一指标爆发增长 | hockey stick曲线 |
| 关键时刻/反转点 | 大数字+K线图 |

**图表数量**：通常 5-8 张，包含 1 张封面图；HTML 卡片模式通常输出 1 张主卡片，也可按用户要求拆成多张。

切分方案确认后，询问用户选择输出模式；如果用户已经明确指定模式，直接进入对应路径。

| 模式 | 适用场景 | 产出 |
| ---- | -------- | ---- |
| AI gen 模式 | 要 LatePost 风格图片、封面图、视觉表现优先 | image-2/gpt-image-2 prompt；用户确认后生成图片 |
| HTML 卡片模式 | 要数字精确、文字可控、可复用网页卡片 | seaborn 图表 + 固定 900×1200px HTML 卡片 |

### 阶段四：通用设计规范

所有图表遵循统一规范：

#### 尺寸与比例

- 3:4竖图（900×1200px）
- 适配手机屏幕阅读

#### 配色方案（根据主题调整，以下为双方对比类默认配色）

- 背景：米白 #F8F7F4
- 主体A：正红 #D62828（或品牌色）
- 主体B：纯黑 #1A1A1A（或对比色）
- 辅助文字：深灰 #555

#### 字体层级

- 主标题：粗体，最大
- 副标题：常规，灰色
- 数据数字：粗体，突出
- 标注/来源：小字，灰色
- 确保同一组输出的字体保持一致

#### 风格约束

- 平面设计，无渐变无阴影无3D
- 无logo，无水印
- 中文财经媒体风格
- 数据来源标注在底部

#### 封面图（可选）

- 概念抽象风格
- 噪声渐变纹理（grainy gradient）
- 高端editorial aesthetic

### 阶段五 A：AI gen 模式

用于输出 image-2/gpt-image-2 prompt，并在用户确认后生成图片。

#### Prompt 撰写

```markdown
## 图N：[图表名称]

[图表描述，包含：]
- 尺寸规格
- 背景色
- 标题/副标题内容
- 图表类型和布局
- 具体数据点和标注
- 配色指定（hex色值）
- 脚注/数据来源
- 风格约束（flat design, no gradients等）
```

**prompt语言**：英文（文生图模型对英文理解更好），图表内文字用中文

**关键原则**：
- 数据必须精确，直接写入prompt
- 标注必须清晰，说明数据来源和单位
- 所有数字必须明确单位，包含币种、时间口径、面积/人数/百分比等必要说明
- 所有图表的坐标轴、图例、标题、标注中必须写清单位，避免只写数字
- prompt 中涉及数字时，必须使用完整表达，如 "$12.5 billion revenue in FY2025"、"同比增长 23%"、"单位：亿美元"
- 避免歧义，明确指定位置（top/center/bottom, left/right）

#### Prompt 预览、用户确认与出图

Prompt 完成后，不要直接开始出图；先完成阶段六 A-D 的发布前审查，再把对应的生图 prompt 作为预览交给用户确认。

#### 预览与确认规则

- 每张图单独展示完整prompt，按叙事顺序排列
- 预览末尾必须附加阶段六 E 项的人工核查清单提醒
- 明确询问用户是否确认开始出图
- 用户确认前，不调用图片生成工具
- 用户确认后，直接调用 image-2/gpt-image-2 开始生成图片，不再重复询问、不再重新规划

#### image-2/gpt-image-2 生成规则

- 模型：优先使用 image-2/gpt-image-2
- 按图表叙事顺序逐张生成，通常先生成封面图，再生成数据图表
- 每次调用图片生成时使用对应图表的完整prompt，不省略数据、标题、标注、硬约束和风格约束
- 尺寸优先使用3:4竖图（900×1200px），如果工具只支持固定尺寸，选择最接近的竖图比例
- 如果用户明确只要求生成prompt、不要求出图，则只输出prompt预览和确认问题，不调用图片生成
- 如果用户要求生成指定张数或指定某几张图，只生成对应图片

#### 生成后检查

- 检查图中文字是否尽量清晰、数据是否与prompt一致
- 检查图表比例是否为3:4竖图，视觉风格是否统一
- 如果图片中关键数据、标题或布局明显错误，修改prompt后用gpt-image-2重新生成

#### 迭代修改

用户可能会：

- 要求调整某张图的数据或布局
- 要求增加标注说明
- 要求修改配色或风格

每次修改先输出修改后的prompt预览；用户确认后，继续用gpt-image-2重新生成对应图片。

### 阶段五 B：HTML 卡片模式

用于制作文字和数字更可控的静态 HTML 数据卡片。

#### 硬性规格

- HTML 卡片只能输出 3:4 竖屏，分辨率锁死为 900×1200px。
- HTML 根容器必须固定 `width: 900px; height: 1200px`，不得用自适应尺寸替代。
- 浏览器截图或导出 PNG 必须固定为 900×1200px。
- 即使用户要求公众号首图、长图、横版或其他分辨率，HTML 卡片模式仍只输出 900×1200px；如确需其他尺寸，必须说明这超出 HTML 卡片模式。

#### seaborn 图表工作流

1. **生成图表主体**：必须使用 Python + seaborn/matplotlib 生成图表主体；不要用 HTML/CSS 手画真实数据图。
2. **统一主题**：必须使用统一的 `sns.set_theme(...)`，固定字体、线宽、网格透明度、坐标轴样式和图例样式。
3. **真实比例**：折线、柱状、条形、散点、热力图、小倍数图等必须按真实数据比例绘制，不能为了版式夸张变形。
4. **嵌入卡片**：将 seaborn 输出的 SVG/PNG 图表嵌入 HTML/CSS 卡片，HTML 负责标题、副标题、说明、来源和 logo 占位。

#### 专业色卡

- 基础色固定为：米白背景 #F8F7F4、炭黑主文字 #1A1A1A、辅助灰 #8E8E8A、浅灰 #B6B3AA、LatePost 红 #D62828。
- 次级色只使用低饱和高级色，例如 muted teal #2F6F6D、warm ochre #B98B2F、steel blue #3F6C8F、slate gray #58616A。
- 同一组卡片必须使用同一套色卡；红色只用于最重要结论、最高优先级数据或唯一强调点。
- 禁止使用高饱和荧光色、彩虹色、随机多色 palette 或与财经/科技媒体气质不符的装饰性配色。

#### 自动图表类型选择

| 数据类型 | 优先图表 |
| -------- | -------- |
| 时间序列 | 折线图、面积图、末端标注线图 |
| 排名/类别比较 | 水平条形图、lollipop chart、dot plot |
| 两组对比 | dumbbell chart、slopegraph、分组柱状图 |
| 占比结构 | 100% stacked bar、treemap-like block、横向比例条 |
| 分布 | 箱线图、violin plot、ridge-like small multiples |
| 相关性 | 散点图、气泡图、带注释回归线 |
| 多指标矩阵 | heatmap、小倍数图、scorecard grid |

可以选择比基础预设更高级的图表形式，但必须优先服务数据可读性、准确性和新闻叙事，不为炫技牺牲理解效率。

#### 统一卡片版式

- 同一组卡片必须使用相同字体栈、字号层级、边距系统、图表区域比例和来源位置。
- 标题、副标题、描述、图表、来源固定在一致区域；标题和副标题表达要克制、和谐，不写营销式口号。
- 右下角预留一个 logo sticker 区域，当前保持空白占位，不放真实 logo、不生成临时 logo。
- 卡片整体保持 LatePost 式米白背景、黑灰主信息、红色重点、高留白和底部来源。
- 自检时必须截图或渲染检查关键文字、数字、图表比例、来源、logo 占位和 900×1200px 画布；发现溢出、遮挡、比例错误或风格不统一时先修复再交付。

HTML 卡片模式不调用 image-2/gpt-image-2；如果用户后续想要图片文件，可将 HTML 截图导出为 PNG。

## 输出格式

### AI gen 模式

图表 Prompt 预览：每张图单独一个代码块，按叙事顺序排列。**末尾必须附加阶段六 E 项的人工核查清单提醒，并询问用户是否确认开始用 image-2/gpt-image-2 出图。用户确认后，直接调用 image-2/gpt-image-2 生成对应图片。**

### HTML 卡片模式

交付 HTML 文件路径、截图预览路径和数据来源说明；如生成了 seaborn 中间图表，也交付图表文件路径。

### 阶段六：发布前审查（强制，不可跳过）

**这是 SKILL 工作流的最后一步，所有 prompt 在交付给用户前都必须逐项过一遍下面的审查清单。如果用户已经拿去出图，仍要在 prompt 末尾追加防错说明，提醒用户出图后人工核查。**

#### A. 事实准确性审查

- [ ] 所有数字（市值、营收、估值、份额、涨跌幅、日期）来源单一、可追溯，避免多源混用
- [ ] 实时数据明确时间戳（精确到小时，如"截至 2026.5.27 18:06"）
- [ ] 排名/数量结论与列表一致（避免"14 家"对应实际 15 行的低级错误）
- [ ] 存量（市值）vs 流量（GDP/营收）比较时必须加限定词（"市值"vs"年 GDP"），避免量纲混淆
- [ ] 单位无歧义：中文"万亿"必须配"美元/人民币/欧元"等明确币种
- [ ] 所有数字都明确写出了单位，且单位与原始数据一致
- [ ] 所有图表的标题、坐标轴、图例、标注、脚注中涉及数字的位置都已规定正确单位
- [ ] 百分比、倍数、金额、人数、面积、时间周期等不同类型数据没有混用或漏写单位
- [ ] 历史事件日期、首次/再次等表述与公开记录一致

#### B. 政治敏感性审查（红线，错了会下架/撤稿/追责）

**核心原则**：所有视觉内容必须从中华人民共和国官方立场出发审查，凡涉及主权、领土、民族、宗教、政治制度、历史事件、领导人、地缘冲突的内容，一律采取最克制、最符合大陆主流媒体规范的处理。**当不确定是否敏感时，默认按敏感处理**。

**B.1 主权与领土完整**
- [ ] **一个中国**：台湾、香港、澳门必须作为中国的一部分呈现。涉及台湾的所有表述用「中国台湾」/「中国台湾地区」，绝不单独用「台湾」、"Taiwan"、"R.O.C."、"Republic of China"；香港用「中国香港」，澳门用「中国澳门」
- [ ] **领土争议地区**：南海诸岛、钓鱼岛及其附属岛屿、藏南地区、阿克赛钦等必须按中国官方立场呈现；克什米尔、北方四岛/南千岛群岛、西撒哈拉、戈兰高地等他国争议地区采取中性或回避处理
- [ ] **地图相关**：能避免出现地图就避免。必须出现时——中国大陆地图必须完整包含台湾、南海九段线、藏南、钓鱼岛；世界地图中中国边界必须按官方版本绘制；不得使用任何含"问题地图"的现成模板
- [ ] **国旗与旗帜图形全部禁用**：所有国家/地区标注一律用中文文字（如「美国」「韩国」「沙特」），不得出现任何旗帜 emoji、旗帜图形、近似旗帜的色块组合（如蓝白红条纹易被读作法国旗，蓝白星条易被读作美国旗或台湾青天白日旗）
- [ ] **国家级标签的视觉对等**：涉及中外主体并列时，避免将中国地区性主体（如「中国台湾」「中国香港」）与"美国/日本/韩国"等独立国家以完全对等的视觉级别并列；可通过字号、缩进、归类等方式做出区分，或加注脚说明

**B.2 民族、宗教与人权敏感议题**
- [ ] **新疆、西藏、内蒙古、宁夏**等少数民族地区相关内容必须按官方表述处理，不得引用境外人权报告、不得使用敏感配图（如特定宗教场所与冲突场景并置）
- [ ] **宗教**：涉及伊斯兰、基督教、佛教、藏传佛教等宗教内容时保持中立、不带立场暗示
- [ ] **法轮功、东突、藏独、港独、台独**等任何分裂势力的口号、旗帜、符号、领袖肖像、组织标志一律禁止出现

**B.3 政治制度、领导人与历史事件**
- [ ] **领导人姓名与肖像**：中国现任及历任领导人、外国元首肖像原则上不入图；必须出现时只用已被官方媒体公开使用的正式照片，不得用漫画化、抽象化或可能被解读为不敬的处理
- [ ] **政治制度对比**：不做"民主 vs 威权""自由 vs 管控"等带价值判断的视觉化对比
- [ ] **历史敏感事件**：1949 年前后政权更替、文革、六四、历次政治运动等不入图；近代史涉及日本、列强等议题时采取官方表述
- [ ] **意识形态符号**：纳粹、共产、宗教极端组织等符号一律禁止；红色五角星、镰刀锤子等可能被误读的图形谨慎使用

**B.4 地缘冲突与国际关系**
- [ ] **俄乌冲突、以巴冲突、伊朗-以色列、朝韩、印巴**等正在进行或长期未解决的冲突，使用最中性的表述，不暗示立场，不显示冲突方旗帜或领土主张图
- [ ] **被中国政府制裁/反制裁的国家或实体**：保持事实陈述，不带情绪化处理
- [ ] **国际组织**：联合国、WTO、IMF 等保持正式称谓，不戏谑

**B.5 商业实体的政治属性**
- [ ] **国有企业**：中国大陆国企（中石油、四大行、三大运营商等）的视觉处理保持庄重，不戏谑
- [ ] **军工、航天、关键基础设施**相关实体的标识与产品图谨慎使用
- [ ] **被列入实体清单/不可靠实体清单**的相关公司，事实陈述，不带情绪
- [ ] **跨国公司的中国/台湾/港澳分部**：标注时遵守 B.1 规则（如"台积电南京厂"明确归属，避免"台湾的台积电"这种表述）

**B.6 通用兜底原则**
- [ ] **"政治正确"高于"信息完整"**：宁可少说、不说，绝不错说
- [ ] **境外信源警惕**：使用 BBC、Reuters、纽约时报、华尔街日报等境外媒体的数据或图表时，特别检查其中是否含有不符合中国立场的标注（如"Taiwan"独立国名、问题地图、不当国旗标识等），不可直接搬用
- [ ] **不确定即回避**：任何"我不确定这样标注是否合适"的瞬间，立即停下来确认或采取最保守处理
- [ ] **借鉴对标**：参照人民日报、新华社、央视、环球时报、财新、晚点 LatePost 等大陆主流媒体的视觉规范，不参照境外媒体规范

#### C. 文生图模型出错风险审查（必须在 prompt 中预防）

文生图模型默认基于西方语境数据训练，对中国大陆政治规范的理解极不可靠。即使 prompt 中明确写了「中国台湾」，模型仍可能：

- 生成台湾地区旗帜或英文 "Taiwan"
- 在中国大陆地图中漏画台湾、南海或藏南

#### D. 文生图模型常见生成错误的预防

- [ ] **数字误生成**：在 prompt 中用 `exact text: "$1.342T"` 这种显式锁定，必要时拆成 "1" "." "3" "4" "2" 单字符要求
- [ ] **单位漏生成/错生成**：在 prompt 中明确写出每个数字旁边必须出现的单位，并要求图表轴标签包含单位，如 "Y-axis label must be: Revenue (USD billions)"
- [ ] **柱状图/折线图按比例**：明确要求 "bar heights STRICTLY proportional to values, where tallest is X× the shortest"，并给出每根柱子相对最矮的倍数
- [ ] **中文字符渲染失真**：复杂中文（"沪深300指数"这种）尽量简化或用英文＋小注
- [ ] **颜色错配**：hex 色值写两遍（"coral red #E63946 (highlight color)"），关键色加描述词
- [ ] **元素重叠/缺失**：明确指定每个元素的位置（top-right corner / below the chart / centered above row 12）
