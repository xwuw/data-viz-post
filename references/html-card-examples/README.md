# HTML Card Reference Set

Use these images as the visual baseline for HTML card mode. Before generating a new HTML card set, inspect this folder or at least `montage.png` to match the established style.

## What To Preserve

- Canvas: fixed 900x1200px vertical cards.
- Background: warm white paper/noise background from `active-warm-white-red-paper`, with low-saturation red/pink areas and no gray cast.
- Chart plate: opaque white or warm-white page inset when needed, square corners (`border-radius: 0`), no glass effect, no strong shadow. Do not copy one identical plate size across every card.
- Typography: high-end Chinese editorial title stack for headlines; clean sans stack for body, chart labels, sources, and annotations.
- Palette: black, charcoal, warm gray, and LatePost red as the main system; red is used only for the leading conclusion or highest-priority data.
- Layout: top metadata line, strong title block, controlled subtitle, centered chart area, bottom explanation/source, lower-right empty logo sticker placeholder.
- Grid: no full-card grid or decorative grid frame. Chart interiors may use only very light reference lines when needed.
- Editorial feel: reduce UI-card feeling. Prefer magazine page composition, direct labels, fine rules, asymmetric alignment, and chart-specific proportions.
- Chart choice: prefer sophisticated editorial forms when data supports them, such as lollipop/dot plot, dumbbell, slopegraph, alluvial/Sankey, layered area, streamgraph, radial/rose, heatmap, small multiples, or annotated tile matrix.

## Files

- `montage.png`: quick overview of all nine successful cards.
- `card-01.png` to `card-09.png`: final successful DeepSeek/Hoover white-paper card set.

When a future output diverges from these references, prefer revising the HTML/CSS and matplotlib/seaborn styling before changing the data narrative.
