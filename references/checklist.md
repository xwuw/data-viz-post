# Checklist

## 阶段六：发布前审查（强制，不可跳过）

**这是 SKILL 工作流的最后一步，所有 prompt 在交付给用户前都必须逐项过一遍下面的审查清单。如果用户已经拿去出图，仍要在 prompt 末尾追加防错说明，提醒用户出图后人工核查。**

### A. 事实准确性审查

- [ ] 所有数字（市值、营收、估值、份额、涨跌幅、日期）来源单一、可追溯，避免多源混用
- [ ] 实时数据明确时间戳（精确到小时，如"截至 2026.5.27 18:06"）
- [ ] 排名/数量结论与列表一致（避免"14 家"对应实际 15 行的低级错误）
- [ ] 存量（市值）vs 流量（GDP/营收）比较时必须加限定词（"市值"vs"年 GDP"），避免量纲混淆
- [ ] 单位无歧义：中文"万亿"必须配"美元/人民币/欧元"等明确币种
- [ ] 所有数字都明确写出了单位，且单位与原始数据一致
- [ ] 所有图表的标题、坐标轴、图例、标注、脚注中涉及数字的位置都已规定正确单位
- [ ] 百分比、倍数、金额、人数、面积、时间周期等不同类型数据没有混用或漏写单位
- [ ] 历史事件日期、首次/再次等表述与公开记录一致

### B. 政治敏感性审查（红线，错了会下架/撤稿/追责）

**核心原则**：所有视觉内容必须从中华人民共和国官方立场出发审查，凡涉及主权、领土、民族、宗教、政治制度、历史事件、领导人、地缘冲突的内容，一律采取最克制、最符合大陆主流媒体规范的处理。**当不确定是否敏感时，默认按敏感处理**。

**B.1 主权与领土完整**
- [ ] **一个中国**：台湾、香港、澳门必须作为中国的一部分呈现。涉及台湾的所有表述用「中国台湾」/「中国台湾地区」，绝不单独用「台湾」、"Taiwan"、"R.O.C."、"Republic of China"；香港用「中国香港」，澳门用「中国澳门」
- [ ] **领土争议地区**：南海诸岛、钓鱼岛及其附属岛屿、藏南地区、阿克赛钦等必须按中国官方立场呈现；克什米尔、北方四岛/南千岛群岛、西撒哈拉、戈兰高地等他国争议地区采取中性或回避处理
- [ ] **地图相关**：能避免出现地图就避免。必须出现时——中国大陆地图必须完整包含台湾、南海九段线、藏南、钓鱼岛；世界地图中中国边界必须按官方版本绘制；不得使用任何含"问题地图"的现成模板
- [ ] **国旗与旗帜图形全部禁用**：所有国家/地区标注一律用中文文字（如「美国」「韩国」「沙特」），不得出现任何旗帜 emoji、旗帜图形、近似旗帜的色块组合（如蓝白红条纹易被读作法国旗，蓝白星条易被读作美国旗或台湾青天白日旗）
- [ ] **国家级标签的视觉对等**：涉及中外主体并列时，避免将中国地区性主体（如「中国台湾」「中国香港」）与"美国/日本/韩国"等独立国家以完全对等的视觉级别并列；可通过字号、缩进、归类等方式做出区分，或加注脚说明

**B.2 民族、宗教与人权敏感议题**
- [ ] **新疆、西藏、内蒙古、宁夏**等少数民族地区相关内容必须按官方表述处理，不得引用境外人权报告、不得使用敏感配图（如特定宗教场所与冲突场景并置）
- [ ] **宗教**：涉及伊斯兰、基督教、佛教、藏传佛教等宗教内容时保持中立、不带立场暗示
- [ ] **法轮功、东突、藏独、港独、台独**等任何分裂势力的口号、旗帜、符号、领袖肖像、组织标志一律禁止出现

**B.3 政治制度、领导人与历史事件**
- [ ] **领导人姓名与肖像**：中国现任及历任领导人、外国元首肖像原则上不入图；必须出现时只用已被官方媒体公开使用的正式照片，不得用漫画化、抽象化或可能被解读为不敬的处理
- [ ] **政治制度对比**：不做"民主 vs 威权""自由 vs 管控"等带价值判断的视觉化对比
- [ ] **历史敏感事件**：1949 年前后政权更替、文革、六四、历次政治运动等不入图；近代史涉及日本、列强等议题时采取官方表述
- [ ] **意识形态符号**：纳粹、共产、宗教极端组织等符号一律禁止；红色五角星、镰刀锤子等可能被误读的图形谨慎使用

**B.4 地缘冲突与国际关系**
- [ ] **俄乌冲突、以巴冲突、伊朗-以色列、朝韩、印巴**等正在进行或长期未解决的冲突，使用最中性的表述，不暗示立场，不显示冲突方旗帜或领土主张图
- [ ] **被中国政府制裁/反制裁的国家或实体**：保持事实陈述，不带情绪化处理
- [ ] **国际组织**：联合国、WTO、IMF 等保持正式称谓，不戏谑

**B.5 商业实体的政治属性**
- [ ] **国有企业**：中国大陆国企（中石油、四大行、三大运营商等）的视觉处理保持庄重，不戏谑
- [ ] **军工、航天、关键基础设施**相关实体的标识与产品图谨慎使用
- [ ] **被列入实体清单/不可靠实体清单**的相关公司，事实陈述，不带情绪
- [ ] **跨国公司的中国/台湾/港澳分部**：标注时遵守 B.1 规则（如"台积电南京厂"明确归属，避免"台湾的台积电"这种表述）

**B.6 通用兜底原则**
- [ ] **"政治正确"高于"信息完整"**：宁可少说、不说，绝不错说
- [ ] **境外信源警惕**：使用 BBC、Reuters、纽约时报、华尔街日报等境外媒体的数据或图表时，特别检查其中是否含有不符合中国立场的标注（如"Taiwan"独立国名、问题地图、不当国旗标识等），不可直接搬用
- [ ] **不确定即回避**：任何"我不确定这样标注是否合适"的瞬间，立即停下来确认或采取最保守处理
- [ ] **借鉴对标**：参照人民日报、新华社、央视、环球时报、财新、晚点 LatePost 等大陆主流媒体的视觉规范，不参照境外媒体规范

### C. 文生图模型出错风险审查（必须在 prompt 中预防）

文生图模型默认基于西方语境数据训练，对中国大陆政治规范的理解极不可靠。即使 prompt 中明确写了「中国台湾」，模型仍可能：

- 生成台湾地区旗帜或英文 "Taiwan"
- 在中国大陆地图中漏画台湾、南海或藏南
- 把"中国香港"渲染成英国殖民时期旗帜
- 在标注俄罗斯时画上前苏联国旗或反向
- 把领导人头像渲染成失真或丑化的版本
- 把宗教符号、政治符号生成在不相关位置

**预防性硬约束必须写进每张涉及国家/地区/政治符号/人物的 prompt 末尾**：

```text
CRITICAL CONSTRAINTS (must be strictly followed):
1. NO flag emojis or flag graphics of any kind anywhere in the image. National/regional identifiers must be Chinese text only.
2. Region labels must follow these exact forms: 中国台湾 (for Taiwan-based entities), 中国香港 (for HK-based), 中国澳门 (for Macau-based). Never use "台湾"/"香港"/"澳门" alone, never use "Taiwan"/"Hong Kong"/"Macau"/"R.O.C." in any language.
3. NO maps of any kind unless explicitly required; if required, China's borders must include Taiwan, the nine-dash line in the South China Sea, and Aksai Chin/South Tibet as part of China.
4. NO political symbols, religious symbols, military insignia, or party emblems unless explicitly part of the data being visualized.
5. NO portraits of any political leader, living or historical.
6. NO colored stripe patterns or color block combinations that could be misread as national flags (e.g., red-white-blue tricolors, blue-white-with-stars, red-with-yellow-stars).
7. All English country names like "Taiwan", "China", "USA", "Korea" must NOT appear in the image; use Chinese text 美国/中国/韩国/中国台湾 etc. only.
```

- [ ] 上述硬约束语句已添加到每张图的 prompt
- [ ] 公司名称在 prompt 中优先使用中文全称（"台积电"而非"TSMC"，"中芯国际"而非"SMIC"），减少模型联想到英文国名
- [ ] 涉及人物头像、商标、宗教/政治符号的请求一律改为抽象符号或几何形状
- [ ] **prompt 提交前自查**：通读一遍 prompt，确认其中没有任何会触发敏感生成的措辞（如"Taiwanese company"、"democratic"、"freedom"、"regime"等带立场词）

### D. 文生图模型常见生成错误的预防

- [ ] **数字误生成**：在 prompt 中用 `exact text: "$1.342T"` 这种显式锁定，必要时拆成 "1" "." "3" "4" "2" 单字符要求
- [ ] **单位漏生成/错生成**：在 prompt 中明确写出每个数字旁边必须出现的单位，并要求图表轴标签包含单位，如 "Y-axis label must be: Revenue (USD billions)"
- [ ] **柱状图/折线图按比例**：明确要求 "bar heights STRICTLY proportional to values, where tallest is X× the shortest"，并给出每根柱子相对最矮的倍数
- [ ] **中文字符渲染失真**：复杂中文（"沪深300指数"这种）尽量简化或用英文＋小注
- [ ] **颜色错配**：hex 色值写两遍（"coral red #E63946 (highlight color)"），关键色加描述词
- [ ] **元素重叠/缺失**：明确指定每个元素的位置（top-right corner / below the chart / centered above row 12）

### E. 用户交付前的必备提醒

每次交付 prompt 序列时，**必须在末尾附加以下提醒**，不能省略：

```markdown
⚠️ 出图后人工核查清单：
1. 逐张检查国家/地区标注，确认无任何旗帜图形、无英文国名（特别盯 TSMC 行：必须是「中国台湾」四个中文字）
2. 逐张检查数字是否与 prompt 一致（文生图模型经常把 $1.342T 渲染成 $1.432T 等乱码）
3. 柱状图/条形图按比例核对（最高柱应是最矮柱的 N 倍，可用尺子比对）
4. 中文字符是否完整、无错字、无残缺笔画
5. 如发现任何一处问题，重新生成，不要 PS 修补，不要"凑合用"
```

## 质量检查清单（旧版精简清单，保留作为快速自查参考）

- [ ] 所有数据都标注了来源
- [ ] 所有估值/营收数字都说明了口径（官方融资轮/私募市场/年化营收等）
- [ ] 所有数字和图表都明确规定了单位，且单位正确
- [ ] 配色一致，hex 色值正确
- [ ] 图表之间有叙事逻辑，不是随机堆砌
- [ ] 阶段六的 A-E 五项审查已全部过完
- [ ] 确保所有prompt输出规定了一致的字体
